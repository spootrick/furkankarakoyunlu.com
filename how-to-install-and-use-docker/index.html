How To Install and Use Docker

Furkan Karakoyunlu
02 Jan 2019
docker, docker ce, docker installation, how to install docker, how to use docker, ubuntu 16.04 docker

Introduction

Docker is a containerization platform that packages your app and its dependencies all together in a structure called docker container. In this way, it ensures that your app will work seamlessly in any environment (windows, linux, mac, cloud, etc.) and produce same output independent from the platform that is running on. This tutorial will be for beginners and includes Docker basics.

In this tutorial, I will be using Ubuntu 16.04 server.

Prerequisites

  • Ubuntu 16.04 (64-bit)
  • Non-root sudo user.

Execute Docker Commands Withoud Sudo

By default, docker commands can only be executed by a root user or by a user in the docker group. The docker group is created automatically by the Docker’s installation process.

If you try to run a docker command without sudo or without being in the docker group, you will get an error like this

To avoid putting sudo at the beginning of every docker command, you can simply add your user to the docker group.

Commit Changes

If you don’t want your changes in a container to be lost, you should commit your changes. Uncommitted changes will be lost with docker rm command.

After making any change from the base image commit your change for future use.

When you commit an image, your new image is saved locally. See docker images.
